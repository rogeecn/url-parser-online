Leitfaden zur URL-Struktur

Lernen Sie die Komponenten von URLs kennen und verstehen Sie die Analyse von Uniform Resource Identifiers

Vollständige URL-Struktur

Die vollständige grammatikalische Struktur von URLs und die Bedeutung jeder Komponente

Schema://[Benutzerinformation@]Host[:Port][/Pfad][?Abfrage][#Fragment]
组件 描述 详细信息
Schema (Protokoll)

Gibt das Protokoll an, das zum Zugriff auf die Ressource verwendet wird, wie http, https, ftp, mailto

Häufig verwendete: http(80), https(443), ftp(21), ws(80), wss(443)
Authoritäts-Komponente

Enthält Benutzerinformationen, Hostnamen und Portnummer zur Identifizierung der Ressource

Format: [Benutzerinformation@]Host[:Port], z.B. benutzer:passwort@beispiel.de:8080
Host (Domain/IP-Adresse)

Identifiziert den Server durch Domänenname oder IP-Adresse und ist der wichtigste Teil einer URL

Kann Domäne (beispiel.de), IPv4 (192.168.1.1) oder IPv6 ([::1]) sein
Portnummer

Gibt den Port der Dienst auf dem Server an. Wenn weggelassen, wird der Standardport für das Protokoll verwendet

Bereich: 1 - 65535, häufige Standards: 80 (HTTP), 443 (HTTPS), 21 (FTP)
Pfad-Komponente

Gibt den genauen Speicherort der Ressource auf dem Server an, ähnlich einem Dateisystempfad

Hierarchische Struktur getrennt durch /: /dokumente/anleitung/installation.html
Abfrageparameter

Schlüssel-Wert-Paare, beginnend mit ?, zur Übertragung zusätzlicher Informationen an den Server

Format: ?schlüssel1=wert1&schlüssel2=wert2. Sonderzeichen müssen URL-codiert werden
Fragmentbezeichner

Beginnt mit # zur Navigation innerhalb der Seite und verweist üblicherweise auf bestimmte Abschnitte der Seite

Nur für den Client sichtbar, wird nicht an den Server gesendet, z.B. #abschnitt1
Benutzerinformation

Enthält Benutzername und Passwort zur Authentifizierung. Selten in modernen Anwendungen verwendet

Format: benutzername:passwort@. Wird aus Sicherheitsgründen nicht empfohlen
URL-Codierungsregeln

Sonderzeichen müssen im Prozentformat codiert werden, um eine korrekte Übertragung der URL zu gewährleisten

Leerzeichen → %20, Chinesische Zeichen → UTF-8-Codierung, reservierte Zeichen wie ? → %3F
Relative URLs

Verweise auf Pfade basierend auf der aktuellen Position der Seite, was die URL-Schreibung vereinfacht

./Gleiches Verzeichnis, ../Ein Verzeichnis höher, /Wurzelpfad, //Protokoll-relative, ?Nur Abfrage
URL-Standards

befolgen RFC 3986 und andere internationale Standards zur Einheitlichkeit und Kompatibilität von URLs

Maximale Länge ist üblicherweise auf 2048 Zeichen beschränkt und unterstützt internationale Domänennamen
